# Milestone v1.1-in-progress: Server Maturity

**Status:** ✅ Complete
**Date:** 2026-06-08
**Phases:** 5-6 of 9

## Overview

The v1.1 milestone matured the server-side administration experience and database flexibility. Both Phase 5 (Server Management Tooling) and Phase 6 (Database Provider Matrix) are complete.

**Milestone retired.** Continuing to v1.2 — Client Polish.

## Completed Phases

### Phase 1: Standalone Mode ✅

**Goal:** A user launches the catalog browser with zero external dependencies, selects a folder of assets, and can browse, search, and view metadata.

**Key Deliverables:**
- Avalonia catalog browser with grid, loupe, and compare views
- SQLite-based local asset catalog with full-text search
- EXIF, IPTC, and XMP metadata extraction and display
- Hierarchical keyword tagging, star ratings, color labels, and flagging
- Curated collections independent of disk folder structure
- SHA256 duplicate detection during folder scan

**Status:** All 11 CATA requirements and META-05 delivered.

### Phase 2: Multi-User Foundation ✅

**Goal:** The catalog browser connects to a running broker service over TCP, authenticates, and browses the shared catalog.

**Key Deliverables:**
- TLS transport with self-signed dev cert fallback (T2.1)
- JWT secret via environment variable + quickstart docs (T2.2)
- Authorization middleware on all handlers (T2.3)
- Stable opcode enum routing (T2.5) — fixed string-based MessageType bug (T2.4)
- BrokerClient auto-reconnect with exponential backoff (T2.6)
- Sidebar fully functional in multi-user mode (T2.7):
  - Folder tree with asset counts
  - Keyword hierarchy with parent/child propagation
  - Media format counts (Image, Video, Document, Audio)
  - Metadata categories hierarchy
  - Date taken tree (year → month)
- Graceful shutdown with task observation (T2.9)
- Brute-force protection via LoginRateLimiter (T2.13)
- JWT jti + iat claims for replay protection (T2.14)
- Structured security logging (T2.15)
- EF Core optimistic concurrency with Version token (T2.16)
- AsNoTracking + AsSplitQuery for read queries (T2.17)
- Connection resiliency with EnableRetryOnFailure (T2.18)
- Client-side timeout/retry on SendAsync (T2.20)
- ChangePoller retry with token expiry detection (T2.21, T2.23)
- Connection status UI in MainWindow (T2.22)
- FolderWatcherHostedService with debounced FileSystemWatcher (T2.24)
- Watched folder DB persistence + admin panel UI (T2.25)

**Architecture Review Blockers Resolved:**
- ✅ CRITICAL-4: No TLS → TLS implemented
- ✅ CRITICAL-5: Hardcoded JWT secret → env var required
- ✅ CRITICAL-6: No authorization → middleware on all handlers
- ✅ CRITICAL-8: Raw tag bug → fixed
- ✅ CRITICAL-9: String routing → opcode enum
- ✅ CRITICAL-10: No reconnection → auto-reconnect implemented
- ✅ HIGH-13: Sidebar non-functional → fully wired

**Critical Issues Fixed:**
- Port mismatch (BrokerClient now uses 9100)
- Static mutable JWT key (instance field)
- TCS memory leak in BrokerClient

### Phase 3: Multi-User Concurrency ✅

**Goal:** Multiple users access the broker simultaneously with consistent state and real-time change propagation.

**Key Deliverables:**
- Connection Registry — per-connection tracking, thread-safe broadcast
- Change Notifications — push-based real-time updates, fire-and-forget broadcast
- Conflict Resolution — ExpectedVersion optimistic concurrency
- Client Refresh — gallery auto-refresh on remote change notification

**Status:** All 3 BROK requirements delivered.

### Phase 4: Metadata Management ✅

**Goal:** Full metadata round-trip — ratings, labels, flags, GPS, copyright, XMP write-back, RAW sidecar, image adjustments, and export.

**Key Deliverables:**
- Rating, Label, Flag, GpsLatitude, GpsLongitude, Copyright, Orientation on DigitalAsset
- Client UI with ComboBoxes for Rating/Label/Flag, TextBoxes for Copyright/GPS, dirty tracking + auto-save
- XMP write-back service with embedded (JPEG/PNG/TIFF/WebP) and RAW sidecar (.xmp)
- Read-only file guard with user toast notification
- Image rotation (90/180/270) and flip (horizontal/vertical) with thumbnail regeneration
- Export dialog for JPEG (quality) and TIFF (LZW/None compression) with optional max dimension resize

**Status:** All 4 META and 4 EDIT requirements delivered.

### Phase 5: Server Management Tooling ✅

**Goal:** Mature the ServiceManager as a dedicated admin tool for user management, service lifecycle, and system audits.

**Key Deliverables:**
- **Admin Panel Dashboard** — consolidated view with mode toggle, service status, migration wizard access
- **User & Role Lifecycle** — full CRUD (add/edit/deactivate users, role picker, validation) in UserManagementView
- **Native Service Installation** — Windows (SCM with UAC elevation + firewall), Linux (systemd), macOS (launchd)
- **Service Configuration** — host/port, TLS settings, polling interval, minimize-to-tray
- **Auto-Refresh Timer** — background status polling configurable 1-300s
- **Audit Log Viewer** — Action/EntityType/Date range filters, standalone DB querying, operation log
- **Cross-Platform Service Installers** — all three platforms fully implemented (not stubs)

**Status:** All 4 ADMIN requirements delivered. 156 ServiceManager tests.

### Phase 6: Database Provider Matrix ✅

**Goal:** Validate and optimize the multi-provider backend for production environments.

**Requirements:** DB-02, DB-03, DB-04 — all ✅

**Key Deliverables:**
- **Provider-Aware Schema**: `AppDbContext.OnModelCreating` generates provider-correct SQL for filtered queries (`NOT IsDeleted`, `"IsDeleted" = FALSE`, `[IsDeleted] = 0`)
- **Provider-Aware Migrations**: `MigrationRunner.MigrateSchemaAsync` uses provider-correct ALTER TABLE quoting
- **Docker Integration Tests**: `DockerAvailability` + `DockerFactAttribute` — auto-run/skip based on Docker presence
- **Configuration**: Default `DbProvider: "sqlite"` and `DbConnection` in `appsettings.json`
- **Provider Config Validation**: Theory test covering all three EF Core provider registrations

## Key Decisions

| Decision | Outcome |
|----------|---------|
| Manual protobuf serialization | Confirmed — no protoc generator in use |
| TLS required for multi-user | Implemented with dev self-signed cert fallback |
| Opcode enum for routing | Values 1-99 allocated, 100+ reserved |
| Dual-mode architecture | Standalone first, multi-user via BrokerClient |
| ServiceManager standalone DB access | ModeManager with configurable provider, not BrokerClient |
| Audit log query for ServiceManager | Direct SQLite query in standalone, no broker dependency |
