## Key Accomplishments

### Phase 18 — Integration: Plugin System for Metadata Extractors

1. **IMetadataExtractor Interface** — Contract with `Priority`, `CanExtract`, `ExtractTextAsync`, `ExtractAsync`. Modeled after the existing `IThumbnailExtractor` pattern for consistency.

2. **ImageExtractor Adapter** — Wraps `MetadataExtractorService` as Priority 100. Uses `HasAnyContent` to return null when no metadata found, enabling priority chain fallthrough.

3. **OfficeExtractor Adapter** — Wraps `OfficeDocumentExtractor` as Priority 200. Text extraction only (office docs don't have rich EXIF/XMP profiles).

4. **HasAnyContent Properties** — Added to `ExtractedTextMetadata` and `MetadataProfile`. Returns true when any field is populated, enabling extractors to return null for empty results so the pipeline continues.

5. **PluginLoaderService** — Core discovery mechanism:
   - Built-in extractors always registered first (Priority 100/200)
   - Plugin directory auto-created on first run (`%APPDATA%/Adam/plugins/`)
   - Assembly scanning via `Assembly.LoadFrom()` with graceful degradation
   - Priority-sorted extraction pipeline with per-extractor error isolation
   - Thread-safe lazy loading (`volatile bool`)

6. **Ingestion Pipeline Integration** — Replaced `new MetadataExtractorService()`/`new OfficeDocumentExtractor()` with `PluginLoaderService` across 6 files:
   - `IngestionViewModel.cs`
   - `FolderWatcherService.cs`
   - `FolderWatcherHostedService.cs`
   - `FolderScanService.cs` (was missing from original plan)
   - `FileIndexer.cs` (already DI-injected, replaced parameter type)
   - `App.axaml.cs` (DI registration)

7. **Sample Plugin Project** — `plugins/SampleMetadataPlugin/` with `SampleVowelCounterExtractor` (Priority 1000) that counts vowels in text files. Full README with plugin development guide.

8. **Plugin Manager UI** — Standalone dialog showing built-in + plugin extractors with:
   - Plugin directory path display
   - Open Plugin Folder button
   - Refresh button to reload plugins
   - Built-in vs third-party section headers
   - Status indicators for each plugin

9. **43 New Tests** (all passing):
   - 7 ImageExtractor tests (CanExtract, delegation, HasAnyContent edge cases)
   - 6 OfficeExtractor tests (CanExtract with extensions, delegation)
   - 10 PluginLoaderService tests (no plugins, valid plugin, corrupt assembly, missing directory, priority, extract all)
   - 8 PluginManagerViewModel tests (load state, refresh, open folder)

## Key Decisions

1. **Interface over abstract class**: `IMetadataExtractor` is an interface for maximum flexibility. No base class inheritance required.

2. **Assembly.LoadFrom over AssemblyLoadContext**: Simple loading for v1. Isolation layer can be added later if plugin conflicts arise.

3. **No dynamic compilation**: Pre-compiled assemblies only. No script-based or runtime-compiled plugins.

4. **Priority convention**: Built-in extractors at 100/200. Plugins at 1000+. Ensures built-ins win for supported types while plugins handle gaps.

5. **Two-phase extraction**: Separate `ExtractTextAsync` (Title/Description/Keywords) and `ExtractAsync` (rich EXIF/XMP profile) — matches existing codebase patterns.

6. **Plugin directory auto-creation**: No error if directory is missing — system runs with built-in extractors only.

7. **Standalone Plugin Manager dialog**: Not a settings tab (since `SettingsViewModel.cs` doesn't exist).
