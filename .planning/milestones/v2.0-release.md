## What's New in v2.0

### Phase 10 — Sidebar CRUD & Tree Interaction

**Goal:** Complete sidebar tree interaction — **XAML ContextFlyout** context menus, cascade delete, inline rename, broker-side CRUD with ChangeNotification, visual filter state.

- **XAML ContextFlyout context menus** on all tree nodes (Keywords, Categories, Collections, Folders) with dynamic code-behind population — Create, Rename, Delete, "Filter by this", "Clear filter", and folder-specific (Reveal in Explorer, Re-scan Folder) entries
- **Inline rename** via F2 or double-click with Enter-to-commit/Escape-to-cancel — committed via `RenameCompletedCommand` wiring in `MainWindow.axaml.cs`
- **Cascade delete** with confirmation dialog showing child count — recursively deletes all descendants in a single transaction (both standalone and multi-user)
- **Visual filter state** — active filter nodes show bold blue `IsActiveFilter` indicator; tracks via `_previousActiveFilterNode` field that handles standalone nodes outside the tree-walk path
- **Filter commands** — `FilterByThisCommand` / `ClearFilterCommand` set/reset the appropriate `Selected*` property
- **Broker CRUD handlers** — cascade delete + ChangeNotification broadcast added to Keyword, Category, and Collection handlers
- **Keyboard shortcuts** — F2 (rename), P/X flags, Enter (reveal), Ctrl+F (search focus)
- **41 SidebarCrud tests** — all passing

### Phase 11 — Full-Text Search (FTS5 / tsvector / CONTAINS)

**Goal:** Complete FTS integration — wire `IFtsService` implementations into `SearchService`, add dedicated search bar with autocomplete suggestions, search result highlighting in gallery tiles, multi-provider validation.

- **`IFtsService` strategy interface** — `SearchAsync`, `GetSuggestionsAsync`, `IsAvailableAsync`, `EnsureReadyAsync`, `RebuildIndexAsync`
- **`SqliteFtsService`** — FTS5 virtual table with rowid mapping, bm25() relevance ranking, prefix matching, phrase queries, sync triggers (INSERT/UPDATE/DELETE on DigitalAssets and AssetKeywords), compile-time detection via `sqlite3_compileoption_used('ENABLE_FTS5')`
- **`PostgresFtsService`** — tsvector/tsquery with GIN index
- **`SqlServerFtsService`** — full-text catalog with CONTAINS
- **`SearchService` dual-path** — uses `IFtsService` when available, falls back to LIKE-based search gracefully
- **Search bar UI** — dedicated search TextBox above the gallery with 300ms debounce, Ctrl+F shortcut
- **Search suggestions** — Popup dropdown with autocomplete from `GetSuggestionsAsync`
- **Search result highlighting** — `IsSearchHighlighted`/`MatchedFieldsText`/`HighlightText` properties on AssetListItem; bold blue match highlighting in tile title; matched-fields badge
- **`TestSearchAsync`** — 29 FTS tests covering all operations, ranking, suggestions, fallback

### Phase 12 — Performance Optimization

**Goal:** Optimize thumbnails, virtualization, and startup for 100K+ asset collections.

- **Decode-to-size** (`ImageThumbnailExtractor.cs`) — uses `DecoderOptions.TargetSize` to constrain image decode to `maxSize`, avoiding full-decode of large source files. ImageSharp handles the downsampling during decode (far less memory + CPU). Safety guard resizes if needed post-decode.
- **Thumbnail cache timestamps** (`ThumbnailService.cs`) — compares `LastWriteTimeUtc` of source vs thumbnail before regeneration, skips regenerating up-to-date thumbnails. Wrapped in try/catch for permission/network failures.
- **Bitmap lifecycle management** (`AssetListItem.cs`) — implements `IDisposable` with `CancelPendingLoad()` (CancellationTokenSource), old bitmap disposal in `Thumbnail` setter with `ReferenceEquals` guard. `Dispose()` removes from shared cache before disposing bitmap.
- **Collection cleanup** (`AssetGalleryViewModel.cs`) — `Dispose()` method; `Assets.Clear()` in search and load paths now disposes items before clearing.
- **Startup profiling** (`ModeManager.cs`, `MainWindowViewModel.cs`) — phase-gated Stopwatch telemetry: `[PERF]` log messages for DB init, sidebar load, gallery load, and total startup time via `ConnectionDebugLogger`.

## Key Decisions Made

| Decision | Rationale |
|----------|-----------|
| XAML ContextFlyout with code-behind population (not `x:DataType`) | Polymorphic node types (KeywordNode, CategoryNode, CollectionNode, FolderNode) can't use compiled bindings on a single MenuFlyout |
| Cascade delete always (no re-parent option) | Simplest, most predictable; user confirms via ConfirmationDialog |
| Dual-path SearchService (FTS when available, LIKE fallback) | Handles FTS5-not-compiled SQLite builds and graceful degradation |
| 300ms debounce on search input | Balances responsiveness against unnecessary FTS queries |
| Decode-to-size via ImageSharp `DecoderOptions.TargetSize` | SkiaSharp doesn't support decode-to-size natively for all formats; ImageSharp handles it in the decoder |
| `_previousActiveFilterNode` tracking for IsActiveFilter | Tree-walk (`ClearActiveFilterStates()`) doesn't reach standalone nodes when `LoadAsync()` hasn't populated tree collections |
| BufferFactor / SkiaOptions / Compiled Bindings skipped | API uncertainty for Avalonia 12 (BufferFactor, SkiaOptions); compiled bindings would break polymorphic templates — best as a dedicated later pass |
