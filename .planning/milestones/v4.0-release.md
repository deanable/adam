## Phase Details

### Phase 19 — Advanced Search & Discovery ✅

**Goal:** Extend the search infrastructure with persistent saved searches, dynamic smart collections, search history, natural language semantic search via ONNX embeddings, and visual similarity search.

**Committed:** `d489c83` — 2026-06-17

**Key Deliverables:**
1. **Saved Searches** — Named, persistable filter+query combinations with pinning; CRUD in broker + standalone
2. **Smart Collections** — Dynamic collections auto-populated from saved search criteria; ✦ indicator in UI; refresh on open
3. **Search History** — Persistent per-user history (last 200 entries) with auto-purge; shown in autocomplete dropdown
4. **Semantic Search (Text)** — Natural language search via `all-MiniLM-L6-v2` ONNX text embedding model (384-dim vectors); cosine similarity in-process
5. **Visual Similarity Search** — "Find Similar" context menu for images; uses LFM2-VL vision encoder pooled output (4096-dim vectors)

**New entities:** `SavedSearch`, `SearchHistoryEntry`, `AssetEmbedding`

### Phase 20 — UX Modernization ✅

**Goal:** Modernize the user experience with a theme engine, full-resolution viewing, asset comparison, and drag-reorder collections.

**Committed:** `b1bc667` — 2026-06-17

**Key Deliverables:**
1. **Theme Engine** — Dark mode variant, 5 accent color presets (Blue, Green, Purple, Orange, Teal), `ThemeService`, settings dialog, persisted to `settings.json`
2. **Full-Resolution Loupe View** — Dedicated full-window viewing mode with `ZoomBorder` control (mouse wheel zoom, click-drag pan, double-click fit-to-window), asynchronous full-res image loading, filmstrip navigation, keyboard shortcuts, info overlay panel
3. **Compare View** — Side-by-side asset comparison with synchronized pan/zoom, overlay/swipe mode with opacity slider, metadata diff table showing differences
4. **Drag-Reorder Collections** — `SortOrder` field on join entity, broker handler for batch reorder, gallery drag-drop with insertion line visual feedback
5. **Design Templates** — `.design_templates/` directory with 5 design reference documents

### Phase 21 — Virtualized Gallery & DB Optimization ✅

**Goal:** Optimize gallery performance with viewport-based virtualization, async thumbnail disk I/O, prioritized batch loading, composite DB indexes, and keyset pagination.

**Committed:** `bcdfd82` / `0ce32ef` — 2026-06-16
**UAT:** All checks passed (`.planning/plans/phase-21/21-UAT.md`)

**Key Deliverables:**
1. **Viewport-Based Virtualization** — `LoadVisibleThumbnails()` calculates visible range with 50% overscan; cancels pending loads for off-screen items
2. **Prioritized Batch Loading** — `BatchLoadRemainingThumbnailsAsync()` loads deferred thumbnails in batches of 8 with 50ms spacing; batch cancelled on navigation
3. **Async Thumbnail I/O** — `FileStream(useAsync: true)` for non-blocking disk reads; LRU memory cache (256 MB); `CancellationToken` throughout
4. **Composite DB Indexes** — 7 composite indexes on `DigitalAsset` for common filter+sort patterns (provider-agnostic via EF Core)
5. **Keyset Pagination** — `ApplyKeysetPagination()` replaces `Skip/TAKE` with seek-based WHERE clause for "load more" calls; supports all 4 sort modes

## Key Decisions

1. **Semantic search over one-shot AI generation**: Text embeddings (semantic search) are preferred over calling an LLM to generate search queries, because embeddings work entirely offline and cost nothing per query.
2. **`all-MiniLM-L6-v2` over larger models**: 384-dim vectors keep storage manageable (1536 bytes per asset for text embeddings). Larger models like `all-mpnet-base-v2` (768-dim) could be added later.
3. **ZoomBorder over ScrollViewer for loupe**: `ScrollViewer` introduces scroll bars that fight with pan gestures. A `RenderTransform`-based approach gives smooth, composable zoom/pan.
4. **No LLM in search**: Natural language search uses embedding similarity, not an LLM. This keeps it offline, fast, and free.
5. **Two-phase v4.0**: Search first (foundation data model), then UX (visual polish). Phases 19 and 20 depend on each other only through the shared `SortOrder` and search filter serialization for smart collections.
6. **Keyset over offset pagination**: Seek-based pagination avoids OFFSET performance degradation at high page numbers; essential for 100K+ asset catalogs.
