## Phase Details

### Phase 19 тАФ Advanced Search & Discovery

**Goal:** Extend the search infrastructure with persistent saved searches, dynamic smart collections, search history, natural language semantic search via ONNX embeddings, and visual similarity search.

**Key Deliverables:**
1. **Saved Searches** тАФ Named, persistable filter+query combinations with pinning; CRUD in broker + standalone
2. **Smart Collections** тАФ Dynamic collections auto-populated from saved search criteria; тЬж indicator in UI; refresh on open
3. **Search History** тАФ Persistent per-user history (last 200 entries) with auto-purge; shown in autocomplete dropdown
4. **Semantic Search (Text)** тАФ Natural language search via `all-MiniLM-L6-v2` ONNX text embedding model (384-dim vectors); cosine similarity in-process
5. **Visual Similarity Search** тАФ "Find Similar" context menu for images; uses LFM2-VL vision encoder pooled output (4096-dim vectors)

**New entities:** `SavedSearch`, `SearchHistoryEntry`, `AssetEmbedding`

### Phase 20 тАФ UX Modernization

**Goal:** Modernize the user experience with a theme engine, full-resolution viewing, asset comparison, and drag-reorder collections.

**Key Deliverables:**
1. **Theme Engine** тАФ Dark mode variant, 5 accent color presets (Blue, Green, Purple, Orange, Teal), `ThemeService`, settings dialog, persisted to `settings.json`
2. **Full-Resolution Loupe View** тАФ Dedicated full-window viewing mode with `ZoomBorder` control (mouse wheel zoom, click-drag pan, double-click fit-to-window), asynchronous full-res image loading, filmstrip navigation, keyboard shortcuts, info overlay panel
3. **Compare View** тАФ Side-by-side asset comparison with synchronized pan/zoom, overlay/swipe mode with opacity slider, metadata diff table showing differences
4. **Drag-Reorder Collections** тАФ `SortOrder` field on join entity, broker handler for batch reorder, gallery drag-drop with insertion line visual feedback
5. **Design Templates** тАФ `.design_templates/` directory with 5 design reference documents

## Key Decisions

1. **Semantic search over one-shot AI generation**: Text embeddings (semantic search) are preferred over calling an LLM to generate search queries, because embeddings work entirely offline and cost nothing per query.
2. **`all-MiniLM-L6-v2` over larger models**: 384-dim vectors keep storage manageable (1536 bytes per asset for text embeddings). Larger models like `all-mpnet-base-v2` (768-dim) could be added later.
3. **ZoomBorder over ScrollViewer for loupe**: `ScrollViewer` introduces scroll bars that fight with pan gestures. A `RenderTransform`-based approach gives smooth, composable zoom/pan.
4. **No LLM in search**: Natural language search uses embedding similarity, not an LLM. This keeps it offline, fast, and free.
5. **Two-phase v4.0**: Search first (foundation data model), then UX (visual polish). Phases 19 and 20 depend on each other only through the shared `SortOrder` and search filter serialization for smart collections.
