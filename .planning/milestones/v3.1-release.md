## Key Accomplishments

### Phase 17 — Collaboration: Asset Comment Threads

1. **Comment Entity Model** — `Comment.cs` with soft-delete (`IsDeleted`), self-referencing parent FK (`ParentCommentId`), concurrency token (`Version`), global query filter `!IsDeleted`, indexes on `AssetId` and `CreatedAt`.
2. **CommentService (Dual-Mode)** — `ListCommentsAsync`, `CreateCommentAsync`, `UpdateCommentAsync`, `DeleteCommentAsync`, `CountCommentsAsync` with standalone (direct DB) and multi-user (TCP broker) paths. Returns tree-structured `CommentDto` sorted chronologically.
3. **Protobuf Wire Messages** — 7 message types (opcodes 120-128) for List, Create, Update, Delete, and real-time push notification (`CommentNotification`). Nested `CommentWire` DTO with self-referencing reply tree.
4. **CommentHandler (Broker)** — Full CRUD with authz (`asset:read` for list, `asset:update` for create/edit/delete), ownership verification (author can edit/delete, admins can delete any), `ChangeNotification` broadcast.
5. **CommentPanelViewModel** — Threaded UI state with `ObservableCollection<CommentThread>`, inline reply/edit modes, delete confirmation, auto-refresh on notification.
6. **CommentPanelView (Avalonia)** — Scrollable threaded layout with relative timestamps, reply count badges (`▶ N replies`), inline reply/editing inputs, empty state, collapsible panel bound to asset selection.
7. **EF Core Migration** — `AddCommentsTable` migration with cascade delete on asset FK, no cascade on self-referencing parent FK or user FK.
8. **Integration Test DI Fix** — Added `CommentHandler` registration to both `BrokerServiceIntegrationTests` and `ConcurrentClientsTests` to resolve 49 integration test failures.

## Key Decisions

1. **Soft-delete with parent retention**: Deleting a comment sets `IsDeleted = true` but retains the body as "[deleted]" to preserve reply thread context. Replies remain visible.
2. **Ownership model**: Author can edit/delete own comments. Admin/Editor role can delete any comment. No admin edit — preserves conversation integrity.
3. **Self-referencing tree**: `ParentCommentId` pointing to the same table. Max nesting depth of 1 (replies to top-level only) — matches GitHub's threading model.
4. **No real-time edit conflict detection**: Last-writer-wins for edits is acceptable given the social nature of comments.
5. **Push notifications via CommentNotification** (opcode 128): Separate from `ChangeNotification` to keep payload small.
6. **Plain text v1**: No Markdown rendering. No email/push external notifications.
