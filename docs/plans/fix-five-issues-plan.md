# Introduction

![Status: Planned](https://img.shields.io/badge/status-Planned-blue)

Five concrete issues identified in the repo review that are causing real risk right now — a committed database file, no schema migrations, domain logic in the DbContext, a denormalized permission model, and zero tests on a service that modifies source files. These are all medium-complexity fixes that don't require changing any public API or UI surface.

---

## 1. Requirements & Constraints

- **REQ-001**: `catalog.db`, `catalog.db-shm`, `catalog.db-wal`, `WindowsTemptest_request.json`, and `query` must be removed from git tracking and added to `.gitignore`.
- **REQ-002**: Schema changes must be managed via EF Core migrations generated by `dotnet ef`, replacing all hand-written `ALTER TABLE` SQL in `ModeManager` and `MigrationRunner`.
- **REQ-003**: `AssociateKeywordsAsync`, `AssociateCategoriesAsync`, and `EnsureKeywordHierarchyAsync` must move out of `AppDbContext` into a new `Adam.Shared/Services/KeywordService.cs`. All 11 call-sites must be updated.
- **REQ-004**: `Role.Permissions` (currently `string[]` serialised as CSV) must be replaced with a `RolePermission` join entity. `AuthorizationMiddleware`, `PermissionEvaluator`, and all tests must be updated.
- **REQ-005**: `MetadataWritebackService` and `MetadataExtractorService` must each have a dedicated unit-test file with ≥ 80% branch coverage of their public methods.
- **CON-001**: The `AppDbContext` public surface must not change in a way that breaks the 383 existing passing tests.
- **CON-002**: All five changes must leave `dotnet test` green (383+ tests passing, 2 Docker-skipped allowed).
- **CON-003**: Each phase is independently committable — no phase depends on an incomplete prior phase being in a "transitional" state.
- **PAT-001**: Follow the `IUiDispatcher` / `SyncUiDispatcher` test infrastructure pattern for any new ViewModel-level tests.
- **PAT-002**: New service classes go in `src/Adam.Shared/Services/`. New tests go in `tests/Adam.Shared.Tests/Services/`.

## 3. Alternatives

- **ALT-001**: For Phase 2, keep `EnsureCreatedAsync` for standalone/test mode and only use migrations for multi-user. **Rejected** — this means the two modes diverge in schema management, which has already caused bugs (e.g., `WatchedFolders` table only in `MigrationRunner`).
- **ALT-002**: For Phase 3, move `AssociateKeywordsAsync` to a static extension method on `AppDbContext` rather than a separate service class. **Rejected** — an extension method on `DbContext` is equally hard to mock and test; a proper service with constructor injection is the right pattern.
- **ALT-003**: For Phase 4, keep `string[]` storage and add a `RolePermission` read model as a view. **Rejected** — a view is non-trivial in SQLite/EF Core and doesn't fix the underlying un-queryable structure.
- **ALT-004**: For Phase 5, use an integration-style HTTP test with a real file on disk rather than a unit test. **Accepted as complement** — the existing `MetadataRoundTripTests.cs` is already integration-style; the new tests are pure unit tests with test-asset files.

## 7. Risks & Assumptions

- **RISK-001**: EF Core migration generation may fail if the `startup-project` (`Adam.BrokerService`) does not have a valid `IDesignTimeDbContextFactory` or `AppDbContext` registered in its DI. Mitigation: add `IDesignTimeDbContextFactory<AppDbContext>` to `Adam.BrokerService` if `dotnet ef migrations add` fails.
- **RISK-002**: Phase 4 (`RolePermission`) is a schema-breaking change. Any database created before this migration will require running `dotnet ef database update` or the app running `MigrateAsync()` on startup. Existing test databases created with `EnsureCreatedAsync()` in tests will not auto-migrate — tests that use `EnsureCreatedAsync()` must switch to `MigrateAsync()` or handle the schema difference.
- **RISK-003**: The 11 call-sites in Phase 3 include ViewModels that resolve `AppDbContext` inside a `using var scope`. Those must resolve `KeywordService` from the same scope, not from the DI root, to avoid scope-lifetime mismatches.
- **RISK-004**: Test JPEG assets committed to git will increase repo size. Mitigation: keep test assets under 50KB each; use Git LFS if the project already uses it.
- **ASSUMPTION-001**: `dotnet ef` global tool version is compatible with EF Core `10.0.0-preview.x`. If not, install the matching preview version: `dotnet tool install --global dotnet-ef --version 10.0.0-preview.3`.
- **ASSUMPTION-002**: The computed `Permissions` property on `Role` (Phase 4 TASK-020) is sufficient to maintain backward compatibility for all read-only access patterns. Write paths that assign `role.Permissions = [...]` will need to be replaced with collection manipulation of `role.RolePermissions`.
