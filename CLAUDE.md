## What This Is

**Adam** is a cross-platform desktop Digital Asset Management (DAM) system built on **.NET 10** and **Avalonia UI**. It operates in two modes from a single codebase:

- **Standalone** — direct SQLite access via `AppDbContext`, no external process.
- **Multi-User** — TCP connection to `BrokerService` with JWT auth against a shared SQLite/PostgreSQL/SQL Server database.

> **`AGENTS.md` is the authoritative, detailed project context** (architecture decisions, phase history, known issues, per-feature notes). Read it for anything not covered here. This file is the quick operational reference.

## Commands

The solution is `Adam.slnx` (XML solution format — use `dotnet build Adam.slnx`, not a `.sln`).

```bash
dotnet build Adam.slnx                 # build everything
dotnet build Adam.slnx -c Release

# Tests — prefer the script: it kills stale testhost processes and runs
# each test project SEQUENTIALLY to avoid DLL lock contention.
./scripts/test.sh                      # all suites
./scripts/test.sh --filter "FullyQualifiedName~AiTagging"
./scripts/test.sh -c Release --skip-cleanup

# Raw dotnet test (one project at a time to avoid parallel-run lock issues):
dotnet test tests/Adam.Shared.Tests
dotnet test tests/Adam.BrokerService.Tests --filter "FullyQualifiedName~DbProvider"

# Run a single test by name:
dotnet test tests/Adam.Shared.Tests --filter "FullyQualifiedName~MyTestMethod"

# Run the apps:
dotnet run --project src/Adam.CatalogBrowser    # desktop client (WinExe)
dotnet run --project src/Adam.ServiceManager    # server admin panel (needs elevation for service ops)
dotnet run --project src/Adam.BrokerService     # TCP broker server

# Publish self-contained binaries (runs tests first unless --skip-tests):
./scripts/publish.sh -r win-x64
./scripts/publish.sh -r linux-x64 --skip-tests
```

### EF Core migrations
The `dotnet-ef` tool is pinned in `dotnet-tools.json` (restore with `dotnet tool restore`). Migrations live in `src/Adam.Shared/Migrations`. The `BrokerService` selects the provider at runtime — see EF notes in `docs/ef-core-migration-guide.md`.

## Project Layout (the three apps + shared core)

| Project | Role |
|---------|------|
| `src/Adam.CatalogBrowser` | **The client** (Avalonia, MVVM). Asset browsing, metadata editing, ingestion, AI tagging, Loupe/Compare views, and *connecting to* a service. No server administration. |
| `src/Adam.ServiceManager` | **The server panel** (Avalonia). 3 tabs: Admin Panel, Users, Audit. Service install/config + user management. |
| `src/Adam.BrokerService` | **The TCP broker server.** Message handlers, transport, runtime DB-provider selection, platform service installers. |
| `src/Adam.Shared` | Code shared by all three: domain models, EF `AppDbContext`, protobuf `Contracts/`, `TcpFrame` transport, services. |
| `src/LiquidVision.Core` | In-repo local **LFM2-VL ONNX** inference for AI image auto-tagging (no Python/cloud). |
| `src/Adam.Generators` | Roslyn source generator: emits protobuf serialization for types marked `[ProtoField]` / `IProtoSerializable`. Referenced as an analyzer. |

**Client-vs-server boundary (strict):** if it manages users, installs services, or configures the server process → `ServiceManager`. If it browses assets, edits metadata, or connects to an existing service → `CatalogBrowser`. Shared logic → `Adam.Shared`.

## Architecture Notes That Cross Files

These are the non-obvious rules that bite if you don't know them. `AGENTS.md` has the full treatment.

- **Transport is raw TCP + length-prefixed protobuf framing** (`TcpFrame`), not HTTP/REST/gRPC. Messages are an `Envelope` (opcode + payload + status + correlation id + auth token) carrying a contract from `Adam.Shared/Contracts/`. Serialization for those contracts is **generated by `Adam.Generators`** from `[ProtoField]` attributes — don't hand-write it, and rebuild after editing a contract. Max payload 256 MB.

- **Database provider is abstracted at runtime** (`DbProviderConfig` in BrokerService). Never hardcode provider-specific SQL or behavior outside the provider classes. Anything touching queries must work on SQLite **and** PostgreSQL **and** SQL Server — `DbProviderMatrixTests` enforces this.

- **SQLite cannot `ORDER BY` a `DateTimeOffset`** (throws `NotSupportedException`, including via `.UtcDateTime` / `EF.Property`). Sort `DateTimeOffset` columns **client-side** after materializing, or use the two-query keyset pattern for pagination. PostgreSQL/SQL Server are fine. See the detailed recipe and the list of refactored locations in `AGENTS.md`; covered by `tests/Adam.Shared.Tests/Data/DateTimeOffsetOrderByTests.cs`.

- **Always implement features for both modes.** A change to asset/metadata behavior usually needs the standalone path (direct `AppDbContext`) *and* the multi-user path (broker message + handler).

- **`AppDbContext.SeedData()` seeds 3 roles** (Viewer/Editor/Administrator) on `EnsureCreatedAsync()`. In tests, don't reuse the names `Viewer`/`Editor` (UNIQUE conflict); use isolated per-test DBs under a temp path and `SqliteConnection.ClearAllPools()` on dispose.

- **ViewModels that hit the UI thread take an optional `IUiDispatcher`** (defaults to `AvaloniaUiDispatcher`). Tests inject a synchronous `SyncUiDispatcher` to avoid `Dispatcher.UIThread` hangs in headless context.

- **AI tagging runs sequentially, never inside `Parallel.ForEachAsync`** — ingest auto-tagging is a post-pass after parallel ingest completes. Registered via `AddLiquidVision()` in `CatalogBrowser/App.axaml.cs`.
